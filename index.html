🧠 How Training Works
Every training example becomes a few-shot prompt for Gemini.

Step 1: Upload a sheet that has been fully reviewed — with correct Environment, Category and Sub Category filled in by a human.

Step 2: The app extracts each row as a ground-truth training example.

Step 3: During every future classification, Gemini receives up to 60 training examples as context, making it learn your team’s exact classification patterns.

Training vs Corrections: Training examples come from bulk uploads. Corrections come from the Review & Correct step. Both are used together.

How AI Learning Works
Every correction you make becomes training data for the AI.

Future Gemini prompts include your last 40 corrections as few-shot examples. The AI learns your team's classification style over time.

Best practice: After each evaluation, spend 5 minutes in Review & Correct to fix mistakes. The system learns quickly and becomes more accurate.

📋 Firestore Security Rules (click to view) 
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Allow any authenticated Firebase user full access
    // This covers both shared/ and users/ paths for all team members
    match /{document=**} {
      allow read, write: if request.auth != null;
    }
  }
}
Copy these rules into Firebase Console → Firestore Database → Rules → Edit rules → Paste → Publish
✓ This allows any signed-in user to read/write Firestore. ⚠ Only share the app with trusted team members.
